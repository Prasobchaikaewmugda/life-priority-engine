# LPE V1.11 Step 3 — Daily Plan Engine Blueprint V1

สถานะ: Design-only / ยังไม่แก้ `app.py`  
เป้าหมาย: ออกแบบระบบสร้าง “แผนวันนี้” จาก `Core Life Profile + Daily Check-in` ให้ไม่ generic และไม่ชนชีวิตจริง

---

## 1. วัตถุประสงค์

Daily Plan Engine ต้องตอบ 5 คำถาม:

```text
1. วันนี้ควรทำอะไร
2. ควรทำเวลาไหน
3. ทำไมต้องทำสิ่งนั้น
4. ถ้าเหนื่อยต้องลดเหลืออะไร
5. อะไรควรเลี่ยงเพื่อไม่พังพรุ่งนี้
```

Engine นี้ยังไม่ใช่ระบบ AI เต็มรูปแบบ และยังไม่ใช่ persistence/database  
MVP ควรเป็น rule-based ก่อน เพื่อควบคุมง่าย ตรวจสอบง่าย และไม่มั่ว

---

## 2. Input ที่ใช้

### 2.1 Core Life Profile

มาจากหน้า `ตั้งค่าชีวิต`

```text
- goals
- time_rhythm
- health_energy
- obligations
- life_rules
```

ใช้เพื่อรู้ว่า:

```text
- เป้าหมายหลักคืออะไร
- เวลาชีวิตปกติเป็นอย่างไร
- งาน/เวรมีข้อจำกัดอะไร
- สุขภาพ/พลังงานควรระวังอะไร
- กฎส่วนตัวหรือสิ่งที่ห้ามทำคืออะไร
```

### 2.2 Daily Context

มาจากหน้า `เช็คอินวันนี้`

```text
- sleep_hours
- energy_level
- mood
- today_shift
- urgent_events
- physical_condition
- today_main_focus
- today_constraints
- today_story
```

ใช้เพื่อรู้ว่า:

```text
- วันนี้พลังจริงเป็นอย่างไร
- วันนี้มีงาน/เวร/เรื่องด่วนอะไร
- วันนี้ควรลดความหนักหรือไม่
- วันนี้เน้นอะไร
- วันนี้มีข้อจำกัดอะไร
```

## 4. หลักการสร้างแผน

### 4.1 กฎพลังงาน

```text
energy 1–2:
- ใช้แผนขั้นต่ำ
- ลดโปรเจกต์หนัก
- อ่านสั้น 10–30 นาที
- ออกกำลังเบาเท่านั้น
- กันเวลาพักและนอน

energy 3:
- ทำงานหลัก 1 อย่างก่อน
- งานรองเป็น optional
- มี fallback ทุกกิจกรรม

energy 4–5:
- ทำงานหลัก + งานรองได้
- ยังต้องไม่ชนเวลานอน
```

### 4.2 กฎการนอน

```text
sleep < 6 ชั่วโมง:
- ห้ามวาง deep work ยาว
- ห้ามวางออกกำลังหนัก
- ลดเป้าหมายให้เป็นขั้นต่ำ
- เพิ่มเวลาพัก

sleep 6–7 ชั่วโมง:
- ทำแผนปานกลาง
- งานใช้สมาธิไม่ควรยาวเกินไป

sleep >= 7 ชั่วโมง:
- ทำแผนเต็มได้ แต่ยังต้องมีพัก
```

### 4.3 กฎงาน/เวร

```text
ถ้ามีเวรเช้า:
- กันเวลานอนก่อนพรุ่งนี้
- งานหนักไม่ควรเริ่มดึก

ถ้ามีเวรดึก:
- กันเวลาพักก่อนเข้างาน
- อย่าวางกิจกรรมหนักก่อนเวร

ถ้าเลิกงานแล้วเหนื่อย:
- เริ่มด้วยพัก/กินข้าว/เดินเบา ก่อนอ่านหรือโปรเจกต์
```

### 4.4 กฎเป้าหมายหลัก

```text
เป้าหมายหลักต้องมาก่อน
แต่ต้องไม่ทำลายสุขภาพ/งาน/การนอน

ถ้าเป้าหมายหลักคืออ่านหนังสือ:
- วางช่วงอ่านเมื่อสมองยังไม่ล้า
- ใช้ block 25–30 นาที
- พัก 5–10 นาที
- ถ้าเหนื่อย ให้ลดเหลืออ่าน 10–15 นาที + สรุป 3 bullet

ถ้าเป้าหมายหลักคือสุขภาพ:
- เริ่มด้วยกิจกรรมที่ทำได้จริง
- ถ้านอนน้อย ให้เน้นเดิน/ยืดเหยียด
- ยังไม่ใช้คำแนะนำแพทย์เฉพาะบุคคล

ถ้าเป้าหมายหลักคือโปรเจกต์:
- เลือก task เล็กที่ปิดได้
- ห้ามเริ่ม deep work ดึกถ้ากระทบนอน
```

## 5. Template แผนวันนี้

ตัวอย่าง output:

```text
แผนวันนี้:
- วันนี้พลังงานต่ำจากการนอนน้อย
- เป้าหมายหลักคืออ่านหนังสือ
- แผนควรเป็นแบบเบาและกันเวลานอน

17:00–17:20 พัก/อาบน้ำ/กินน้ำ
เหตุผล: เพิ่งจบงาน ไม่ควรเริ่มงานหนักทันที
fallback: นอนพัก 15 นาที

17:30–17:50 เดินเบา 20 นาที
เหตุผล: สุขภาพสำคัญ แต่วันนี้ไม่ควรหนัก
fallback: ยืดเหยียดในบ้าน 10 นาที

19:00–19:30 อ่านหนังสือ
เหตุผล: เป้าหมายหลักคือสอบ และยังไม่ดึก
fallback: อ่าน 15 นาที + สรุป 3 bullet

21:30 หยุดงานหนัก
เหตุผล: ป้องกันนอนดึกและพรุ่งนี้พัง

23:00 เข้านอน
เหตุผล: ฟื้นพลังเพื่อวันถัดไป
```

## 6. Missing Information Handling

ถ้าข้อมูลไม่พอ ห้ามเดาแบบมั่นใจ ต้องแสดง:

```text
ข้อมูลที่ยังขาด:
- เวลาตื่น/นอนปกติ
- เวรวันนี้
- พลังงานวันนี้
- deadline ของเป้าหมายหลัก
```

และสร้างแผนแบบ conservative:

```text
- ใช้ block สั้น
- ไม่วางงานหนักดึก
- มี fallback ทุกกิจกรรม
```
